## 3. 部署指南

### 3.1 部署路径

```
https://monster-lair.vn/campaign/streamer-dropin/
```

### 3.2 部署步骤

1. **上传静态资源**
   ```bash
   # 上传 assets/ 到 CDN
   aws s3 sync assets/ s3://cdn.monster-lair.vn/campaign/streamer-dropin/assets/
   ```

2. **上传配置文件**
   ```bash
   # 上传 config.json（支持热更新）
   aws s3 cp config.json s3://cdn.monster-lair.vn/campaign/streamer-dropin/config.json
   ```

3. **部署页面**
   ```bash
   # 上传 index.html
   aws s3 cp index.html s3://monster-lair.vn/campaign/streamer-dropin/index.html
   ```

### 3.3 配置热更新

`config.json` 支持热更新，修改后无需重新部署页面：
- 页面每 5 分钟自动检查配置更新
- 或用户刷新页面立即生效

## 5. 协作流程

### 5.1 问题反馈渠道

| 问题类型 | 负责人 | 反馈方式 |
|----------|--------|----------|
| 业务逻辑疑问 | 产品经理 | PRD.md 评审 |
| 技术实现疑问 | 前端开发 | README.md 查阅 |
| 部署问题 | 运维 | 本文档 + 运维手册 |
| 资源替换 | 设计/产品 | 交付物清单确认 |

### 5.2 变更管理

| 变更场景 | 操作 | 影响范围 |
|----------|------|----------|
| 修改活动时间 | 更新 config.json | 无需重新部署页面 |
| 调整皮肤价格 | 更新 config.json | 无需重新部署页面 |
| 公布阵营结果 | 更新 config.json | 无需重新部署页面 |
| 修改页面文案 | 更新 i18n.json + 重新构建 | 需重新部署页面 |
| 新增埋点事件 | 修改 index.html | 需重新部署页面 |
